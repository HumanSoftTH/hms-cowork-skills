§2

/crm — ข้อมูลลูกค้า

/crm

ค้นหาลูกค้า ดูข้อมูลพื้นฐาน follow-up, incident, ticket, สถานะการชำระ และ stage ใน pipeline

2.1 ตัวอย่างคำถาม — คลิกเพื่อ copy

ค้นหาลูกค้าชื่อ ABC Technology copy
ดูข้อมูลบริษัท domain: example.co.th copy
ดู follow-up ล่าสุดของลูกค้า ABC Technology copy
incident ทั้งหมดของลูกค้า example.co.th copy
ticket เปิดค้างของลูกค้า XYZ มีอะไรบ้าง copy
ค้นหา ticket เรื่องการ export รายงาน copy
สถานะการชำระเงินของ ABC Technology ตอนนี้ copy
ลูกค้า example.co.th อยู่ใน stage อะไรของ pipeline copy
แพ็กเกจและวันหมดอายุของ example.co.th copy

2.2 ฟังก์ชันที่ใช้ได้

ฟังก์ชัน ใช้สำหรับ input ที่ต้องการ
ค้นหาลูกค้า หาจากชื่อบริษัท, domain, หรือชื่อคน คำค้นหา (บางส่วนก็พอ)
ข้อมูลพื้นฐาน แพ็กเกจ, วันหมดอายุ, channel, sizing, ผู้ดูแล ชื่อบริษัทหรือ domain
Follow-up บันทึก follow-up เรียงตามวันที่ล่าสุด พร้อม topic ชื่อบริษัทหรือ domain
Incident รายการ incident พร้อมสถานะและวันที่ ชื่อบริษัทหรือ domain
Ticket ค้นหา ticket หรือดูรายละเอียด ticket ของลูกค้า คำค้น / ticket ID / ชื่อบริษัท
Payment Status สถานะการชำระ, ยอดค้าง, ประวัติ ชื่อบริษัทหรือ domain
Customer Stage stage ปัจจุบัน, วันที่เข้า stage, ประวัติการเปลี่ยน ชื่อบริษัทหรือ domain

เคล็ดลับ: ถ้าไม่รู้ customer ID ให้ค้นหาชื่อก่อน Claude จะแสดงรายการแล้วให้เลือก ถ้าพิมพ์ภาษาไทยไม่เจอ ลองพิมพ์ domain หรือชื่อภาษาอังกฤษ

§5

เคล็ดลับการใช้งาน

  • 01 ไม่ต้องพิมพ์ /skill ก็ได้ — ถามตรงๆ ว่า "ดู funnel เดือนนี้" หรือ "หาลูกค้าชื่อ ABC" Claude เลือก skill ที่เหมาะสมให้อัตโนมัติ /skill ใช้เมื่ออยากบอกให้ชัดเจนว่าต้องการทำอะไร
  • 02 ถาม drilldown ต่อได้ทันที — หลังได้ตัวเลขแล้ว พิมพ์ต่อในช่อง chat เดิม เช่น "ขอดูรายละเอียดของเดือน เม.ย." Claude จะจำ context ไว้และ drilldown ให้
  • 03 บอกบทบาทถ้าต้องการสรุปแบบเฉพาะ — เช่น "สรุปให้ฝ่ายบริหาร" หรือ "ต้องการข้อมูลดิบสำหรับ Excel" จะได้ format ที่ต่างกัน
  • 04 ค้นหาลูกค้า — พิมพ์บางส่วนก็พอ — ระบบใช้ LIKE search ดังนั้น "ABC" จะพบ "ABC Technology", "ABC Group" ฯลฯ ถ้าไม่เจอจากชื่อบริษัท ลองพิมพ์ domain หรือชื่อคนแทน
  • 05 Report ที่ดึง range กว้าง อาจช้า — analytics report แนะนำระบุช่วงไม่เกิน 6 เดือน ส่วน /report ที่ไม่มี date range (เช่น domain หมดอายุทั้งหมด) อาจใช้เวลา 30–60 วินาที
  • 06 ข้อมูลทั้งหมดเป็น real-time จาก ACP production — ไม่มี cache ดังนั้นตัวเลขที่ได้คือข้อมูล ณ เวลาที่ถาม ถ้าต้องการ snapshot สำหรับ report ให้ copy ผลลัพธ์ออกมาเก็บไว้
  • 07 S + M + L = All Size เสมอ — Sizing S = 1–75 คน (รวม domain ที่ไม่ระบุขนาดบริษัท), M = 76–200 คน, L = 201+ คน ถ้าบวกทั้งสามจะได้ All Size
  • 08 เปิดคู่มือนี้ได้ตลอด — พิมพ์ /cheatsheet ใน Claude cowork เมื่อไหรก็ได้ Claude จะเปิดไฟล์นี้ในเบราว์เซอร์ให้
§6

ติดตั้ง Plugin

ขั้นตอนติดตั้ง (ครั้งแรก)

ต้องมี Claude Code CLI ติดตั้งแล้ว รัน 2 คำสั่งนี้ใน terminal:

ลำดับคำสั่ง
1 claude plugin marketplace add https://github.com/HumanSoftTH/hms-cowork-skills
2 claude plugin install hms-cowork-skills

หลัง install เสร็จ restart Claude แล้วใช้งานได้เลย · อัปเดต: claude plugin update hms-cowork-skills
